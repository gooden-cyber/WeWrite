# Product Feature Design Skill

产品功能设计技能。用于设计和规划产品功能，确保用户体验优秀。

## 设计原则

### 1. 用户优先

- **简单易用**：减少用户学习成本
- **即时反馈**：每个操作都有明确反馈
- **容错设计**：允许用户犯错并轻松恢复

### 2. 功能完整性

- **核心功能**：必须完整可用
- **辅助功能**：增强用户体验
- **边界情况**：处理异常场景

### 3. 交互流程

```
用户目标 → 操作路径 → 即时反馈 → 结果确认
```

## 功能设计清单

### 新功能设计

1. **明确用户目标**
   - 用户想做什么？
   - 当前如何完成？
   - 痛点是什么？

2. **设计操作路径**
   - 最少步骤完成
   - 清晰的引导
   - 明确的按钮/链接

3. **即时反馈机制**
   - 加载状态
   - 成功/失败提示
   - 进度指示

4. **边界情况处理**
   - 空状态
   - 错误状态
   - 超时处理

### 交互设计规范

#### 按钮状态

```
默认状态 → 悬停状态 → 点击状态 → 加载状态 → 完成状态
```

#### 反馈时机

| 操作 | 反馈时机 | 反馈方式 |
|------|----------|----------|
| 点击按钮 | 立即 | 按钮状态变化 |
| 提交表单 | 1秒内 | 加载动画 |
| 长时间操作 | 实时 | 进度条/百分比 |
| 操作成功 | 完成后 | 成功提示 |
| 操作失败 | 失败后 | 错误提示 + 解决方案 |

#### 提示信息

```python
# ✅ 好的提示
"文章发布成功！已创建草稿，等待审核。"

# ❌ 不好的提示
"操作完成"
```

## 常见问题

### 问题 1：配置修改不实时生效

**原因**：配置文件只在启动时读取一次

**解决方案**：
1. 每次执行时读取配置
2. 使用文件监听机制
3. 使用数据库存储配置

### 问题 2：用户不知道操作结果

**原因**：缺少反馈机制

**解决方案**：
1. 添加 Toast 提示
2. 添加状态栏
3. 添加通知系统

### 问题 3：操作路径太长

**原因**：功能设计不合理

**解决方案**：
1. 合并步骤
2. 添加快捷操作
3. 提供默认值
